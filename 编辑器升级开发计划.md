## 0. 背景与核心决策

### 开发前现状（留档）

- 源码编辑使用原生 `<textarea>`（`src/App.tsx` 约 1354 行处），无语法高亮。
- `EditorMode = "split" | "preview"`（`src/types/document.ts`），preview 为只读渲染。
- 滚动同步依赖隐藏镜像 div 测量 textarea 行坐标（`App.tsx` 约 760–920 行，`rebuildScrollAnchors` / `syncScroll`），配合 `lib/markdown.ts` 的 `getBlockStartLines`。
- 搜索高亮依赖 `search-backdrop` 垫层 div 模拟（`App.tsx` 分栏编辑区内 + `document-search.tsx`）。
- 工具栏 `markdown-toolbar.tsx` 通过 `textareaRef` 直接操作 `HTMLTextAreaElement`。

### 核心架构决策（不可违背）

1. **单一数据源**：全程只有一份 Markdown 源文本，存于 CodeMirror `EditorState`。"渲染编辑"是源码之上的装饰视图（Obsidian Live Preview 模式），**禁止**引入 HTML→Markdown 反向转换（不用 Turndown、不用 TipTap）。
2. **一个 EditorView 贯穿全部状态**：编辑方式切换只通过 `Compartment.reconfigure` 开关装饰扩展 + CSS 显隐右栏；进入阅读模式时只隐藏、不卸载 EditorView，从而保证光标、选区、撤销历史、滚动位置跨状态保持。切换文档时则必须清空上一文档的撤销历史。
3. **React 状态同步方向**：CodeMirror 为编辑权威，通过 `updateListener` 将文档变更同步到 React 的 `draft.markdown`（用于预览渲染、字数统计、自动保存、大纲）。React → CM 仅在切换文档 / 打开文件时整体替换文档内容。
4. 布局上第一阶段保留现有外层滚动容器（CM 配置为随内容增高），与现有标题输入框布局兼容；虚拟滚动优化留到最后阶段按需处理。

### 新增依赖

```bash
pnpm add @codemirror/state @codemirror/view @codemirror/language @codemirror/commands @codemirror/lang-markdown @lezer/highlight
```

不引入 `@codemirror/search`（沿用自研搜索 UI，改用 CM decoration 实现高亮）。

## 1. 阶段一：基础替换（textarea → CodeMirror）✅

### 任务

1. 新建 `src/components/markdown-editor.tsx`：封装 CodeMirror 的 React 组件。
   - Props：`value`、`onChange(markdown: string)`、`placeholder`、暴露 `EditorView` 的 ref（供工具栏/搜索/滚动同步使用）。
   - 内部用 `useRef` 持有 `EditorView`，`useEffect` 创建/销毁；`value` 外部变化时（切换文档）用 `dispatch` 整体替换并重置光标到文档头。
   - 预置 `Compartment`：`themeCompartment`、`livePreviewCompartment`（本阶段为空，后续阶段填充）。
   - 基础扩展：`history()`、`keymap`（`defaultKeymap` + `historyKeymap`）、`EditorView.lineWrapping`、placeholder 扩展。
2. 在 `App.tsx` 分栏模式中用该组件替换 `<textarea>`，删除 textarea 自适应高度的 `useLayoutEffect`（约 760–766 行）。
3. 样式对齐：字号 14px、行高 1.9、透明背景、无边框无 outline，最大宽度 42rem 版心不变。CM 主题通过 `EditorView.theme` 读取现有 CSS 变量（`--foreground`、`--muted-foreground` 等），确保 light/dark/system 三主题正常。
4. `draft.richHtml` 字段维持现状不动（后续阶段清理）。

### 验收标准

- 分栏模式下可正常输入、粘贴、撤销/重做（Cmd+Z / Cmd+Shift+Z）。
- 中文输入法组合输入（拼音候选）无异常、无重复字符。
- 切换文档、新建文档、打开外部 .md 文件内容正确加载。
- 自动保存、字数统计、大纲提取、导出 HTML 均正常（它们只依赖 `draft.markdown`，不应感知底层变化）。
- 三种主题下编辑区配色正常。
- `pnpm lint` 与 `pnpm build` 通过。

## 5. 阶段五：三种编辑方式与阅读模式 ✅

> 本阶段工作量最大，按子步骤依次提交，每步独立可验证。

### 5a. 模式与开关骨架

- `EditorMode` 扩展为 `"source" | "split" | "live" | "preview"`（`src/types/document.ts`）。界面文案分别为“源码 / 对照 / 即时 / 阅读”。四个文字按钮可直接切换，前三者属于编辑方式，阅读为独立只读状态。
- `livePreviewCompartment.reconfigure` 在 live 模式注入装饰扩展集，split 模式注入空集。
- App 布局：source 与 live 为单栏 CM（版心与预览一致，42rem），split 保留现有双栏；preview 复用当前预览页面作为阅读模式。
- 编辑方式使用选择菜单；阅读模式使用独立按钮。`Cmd+E` 进入/退出阅读模式，退出后返回此前使用的编辑方式。
- 工具栏在 source / split / live 中显示，在阅读模式隐藏；从阅读模式发起替换时先返回此前编辑方式。

### 5b. 行内格式装饰

新建 `src/lib/live-preview.ts`，实现基于 `syntaxTree` 遍历的 `StateField<DecorationSet>`：

- 粗体/斜体/删除线/行内代码/高亮：内容应用对应样式（复用阶段二的高亮样式），语法标记（`**`、`` ` `` 等）在**光标不在该节点范围内时**用 `Decoration.replace` 隐藏；光标进入（选区与节点范围相交）时还原显示。
- 链接 `[text](url)`：光标在外时只显示 text 并带链接样式，Cmd+点击打开链接；光标进入时显示完整源码。
- 标题 `#` 前缀：光标不在该行时隐藏，标题文字按阶梯字号渲染。

### 5c. 块级装饰

- **数学公式**：`$$...$$` 块与 `$...$` 行内，光标在外时用 `Decoration.replace` + `WidgetType` 渲染 KaTeX 结果（复用 `ensureKatex`）；点击 widget 或光标进入时还原源码。渲染失败（语法错误）显示源码并加错误色。
- **图片** `![alt](src)`：光标在外时在该行下方渲染图片 widget（本地相对路径按 vault 路径解析，可复用预览侧已有逻辑；若预览侧无此逻辑则先只支持 http(s) 与绝对路径，标注 TODO）。
- **代码块**：不做整块替换，保留源码 + 等宽字体 + 整块淡背景；围栏标记 ``` 光标在外时弱化显示（降透明度，不隐藏）。内嵌语言高亮标注 TODO。
- **引用块/列表/分隔线/任务列表**：用 `Decoration.line` 加样式（引用左边线、列表缩进、`---` 光标在外时渲染为水平线、`- [ ]` 渲染为可点击 checkbox——点击切换 `[ ]`/`[x]` 需 dispatch 对应文本修改）。
- **表格**：v1 不做渲染，保留源码 + 等宽字体，标注 TODO。

### 5d. 交互打磨

- "光标所在行/块显示源码"的判定统一封装（选区与节点范围相交即揭示），确保多光标、整段选中时行为合理。
- 撤销/重做、全选复制在 live 模式下操作的是源码文本（decoration 不影响 doc）。
- 性能：需要依赖 `view.visibleRanges` 的装饰使用 `ViewPlugin` 计算；仅与文档有关的状态才放入 `StateField`。

### 验收标准

- live 模式下打开《Markdown语法测试文档.md》：标题、粗斜体、行内代码、链接、公式、图片、引用、列表、任务列表、分隔线均按渲染效果显示；光标移入任意元素时该处还原为源码可编辑，移出后恢复渲染。
- 三种编辑方式与阅读模式切换：文档内容、光标位置、撤销历史全部保持；来回切换 20 次无状态错乱、无内存泄漏迹象（EditorView 不重建）。

| 风险 | 预案 |
|------|------|
| 中文 IME 在 decoration 边界的组合输入异常 | CM6 对 IME 支持成熟，但需在 5b/5c 每步实测；若某类 replace decoration 触发问题，改为光标所在整行禁用装饰（Obsidian 同款策略已是整行揭示，天然规避大部分场景） |
